GridFooter: Layout & Position

Where should the footer sit and what should it look like?

A

Sticky footer (always visible)

Footer sticks to the bottom of the grid container, always visible even while scrolling through data.

Pros

  • User always sees count info
  • Clear visual boundary for the grid

Cons

  • Takes vertical space permanently
  • With infinite scroll, "page" concept is odd
B

Inline footer (scrolls with content)

Footer sits at the bottom of the data, after the last row. Scrolls naturally with the content. On infinite scroll grids, it updates as more data loads.

Pros

  • Doesn't eat vertical space
  • Natural for infinite scroll — sits before the sentinel

Cons

  • Not visible until user scrolls near the end
C

Sticky footer — minimal (count only, no page info)

Sticky like A, but slimmer — no "Página X de Y" since this is an infinite scroll grid, not traditional pagination. Just the count.
