GridFooter: Entity Label Strategy

How should the footer describe the items being shown?

A

Per-entity label via input

Each consumer passes an entityName input like "produtos", "clientes", "pedidos".

Customers page
Mostrando 20 de 147 clientes Página 1 de 8
Products page
Mostrando 20 de 1.250 produtos Página 1 de 63

Pros

  • Contextual — feels natural in Portuguese
  • Each page reads correctly

Cons

  • Every consumer must pass the label
B

Generic "registros" default + optional override

Footer defaults to "registros" but consumers CAN override with entityName if they want.

Without override (default)
Mostrando 20 de 147 registros Página 1 de 8
With override: entityName="produtos"
Mostrando 20 de 1.250 produtos Página 1 de 63

Pros

  • Works out of the box with zero config
  • Consumers can opt-in to specific labels
  • Best of both worlds

Cons

  • "registros" is slightly less specific than the entity name
C

No entity label — just numbers

Footer only shows counts, no noun at all.

Any page
Mostrando 20 de 147 Página 1 de 8

Pros

  • Zero config, always correct

Cons

  • Less informative — "20 de 147" what?
  • Feels incomplete in Portuguese
