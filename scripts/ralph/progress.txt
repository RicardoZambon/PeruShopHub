## Codebase Patterns
- **Redis queue pattern**: Use `IConnectionMultiplexer` + `ListLeftPushAsync` for Redis LIST operations (queues). `IDistributedCache`/`ICacheService` only supports key-value, not lists
- **Webhook skip paths**: New unauthenticated endpoints must be added to skip lists in BOTH `TenantMiddleware` AND `TenantRateLimitMiddleware`
- **Marketplace adapter pattern**: `IMarketplaceAdapter` in Core/Interfaces, implementations in Infrastructure/Marketplace. Keyed DI: `AddKeyedScoped<IMarketplaceAdapter, XAdapter>("key")`. Named HttpClient per marketplace
- **ML API DTOs**: Use `[JsonPropertyName("snake_case")]` on ML response models — ML API uses snake_case, not camelCase
- All components are **standalone** (no NgModules) with `standalone: true`
- Use `CommonModule` import for `*ngFor`, `*ngIf` directives
- Barrel exports in `shared/components/index.ts` — export both component and types
- SCSS uses `@use '../../../../styles/mixins' as m;` for responsive breakpoint mixins
- CSS custom properties from `tokens.css` for all design tokens (e.g., `var(--surface)`, `var(--neutral-200)`)
- IntersectionObserver for infinite scroll: run outside NgZone, use scroll container as root, re-enter zone only to emit events
- Angular build via `npx ng build --configuration=development` — takes ~95 seconds
- Template file convention: separate `.html` file with `templateUrl`
- Style file convention: separate `.scss` file with `styleUrl` (singular, not `styleUrls`)
- **Service layer pattern**: All business logic in Application/Services, controllers are thin HTTP adapters
- **Exception pattern**: NotFoundException (404), AppValidationException (400 with errors dict), ConflictException (409) — caught by GlobalExceptionFilter
- **Validation pattern**: Collect all errors into `Dictionary<string, List<string>>` before throwing AppValidationException
- **Optimistic locking**: Version property + ConcurrencyToken on Product, Category, PurchaseOrder, Supply — set OriginalValue, increment, catch DbUpdateConcurrencyException
- **Role-based auth**: Admin for Settings, Admin/Manager for mutations, any authenticated for reads
- **Build command**: `dotnet build src/PeruShopHub.API/PeruShopHub.API.csproj /nodeReuse:false /p:UseSharedCompilation=false -o /tmp/perushophub-build` (VS Code locks default bin/)
- Application project needs explicit EF Core 9.* package reference due to version resolution issues
- BCrypt.Net-Next already referenced in Infrastructure project
- Product interface defined inline in `product.service.ts`, Category in `models/category.model.ts`, Supply in `supply.service.ts`, PO in `purchase-order.service.ts`
- Error interceptor uses functional `HttpInterceptorFn` pattern (not class-based)
- Remote is `RicardoZambon/PeruShopHub` on GitHub — use for badge URLs and ghcr.io paths
- **Test infrastructure**: Unit tests in `tests/PeruShopHub.UnitTests/` (xUnit + FluentAssertions + Moq), integration tests in `tests/PeruShopHub.IntegrationTests/` (xUnit + TestContainers)
- **Integration test pattern**: Use `[Collection("Integration")]` attribute + `IntegrationTestBase` base class. `CustomWebApplicationFactory` provides PostgreSQL 16 + Redis 7 via TestContainers, auto-migrates DB
- **Program class accessibility**: `public partial class Program { }` at end of `Program.cs` enables `WebApplicationFactory<Program>` in tests
- **TestContainers builder pattern**: Use `new PostgreSqlBuilder("postgres:16-alpine")` (image in constructor, not `.WithImage()`)
- **Unit test DB pattern**: Use SQLite in-memory (`SqliteConnection("DataSource=:memory:")`) with `EnsureCreated()` for unit tests. Provide `ITenantContext` mock with `IsSuperAdmin=true` to bypass query filters
- **SQLite + EF Core gotcha**: SQLite DELETE in batch with INSERT/UPDATE on same table causes `DbUpdateConcurrencyException`. Workaround: test delete logic separately or test the underlying calculation methods directly
- **CostCalculationService testing**: Service uses DbContext directly (not repository), so tests need real DB (SQLite in-memory). Mock `INotificationDispatcher` and `ILogger`. Use `IConfiguration` in-memory collection for tax rate
- **SQLite LINQ limitation**: Complex projections with `.Select()` containing subqueries (e.g., `Variants.Sum(v => v.Stock)`) followed by `.OrderBy()` on the projected result fail with SQLite. These queries need integration tests with PostgreSQL
- **Integration test auth pattern**: Register via `POST /api/auth/register` → extract AccessToken from AuthResponse → set `Client.DefaultRequestHeaders.Authorization = new AuthenticationHeaderValue("Bearer", token)` → all subsequent requests are authenticated with tenant context
- **Integration test data seeding**: Orders (and other webhook-sourced entities) must be seeded via `CreateDbContext()` since there's no POST endpoint. Set TenantId from the auth response's `User.TenantId`
- **Moq expression tree gotcha**: When verifying methods with optional `CancellationToken` params, you must include `It.IsAny<CancellationToken>()` explicitly — Moq expression trees can't handle optional arguments
- **PurchaseOrderService testing**: AddCost/UpdateCost/RemoveCost with Items involve INSERT+UPDATE batches that can trigger SQLite concurrency issues. Test validation paths and simpler operations; leave full CRUD cost tests for integration
- **Angular testing framework**: Vitest + `@analogjs/vite-plugin-angular` (not Karma/Jasmine). Config at `vitest.config.ts`, tests use `*.spec.ts` convention
- **Angular TestBed init**: vitest `setupFiles` doesn't share module context with test files — import `test-setup.ts` directly in each spec file that uses TestBed
- **Angular TestBed reset between files**: When tests run after other test files, call `TestBed.resetTestingModule()` + `ensureTestBedInit()` in `beforeEach` to avoid "module already instantiated" errors
- **jsdom missing APIs**: Mock `IntersectionObserver` with `vi.stubGlobal()` — jsdom doesn't provide it
- **HttpTestingController URL matching**: In jsdom, relative URLs like `/api/auth/login` resolve to `http://localhost:5000/api/auth/login`. Use `req => req.url.endsWith('/auth/login')` predicate instead of string match
- **Angular test script**: `npm test` runs `vitest run` (not `ng test`). CI uses `npx vitest run`
- **Rate limiting middleware**: `TenantRateLimitMiddleware` uses `IDistributedCache` (Redis) for per-tenant counters. Placed after TenantMiddleware in pipeline. Config in `RateLimiting:RequestsPerMinute`
- **Serilog logging**: Structured JSON logging via Serilog. Config in `Serilog:` section of appsettings.json. `CorrelationIdMiddleware` first in pipeline, `RequestLoggingMiddleware` after auth. Use `LogContext.PushProperty()` to enrich logs
- **Sentry error tracking**: `Sentry.AspNetCore` 6.x via `builder.WebHost.UseSentry()`. DSN from `SENTRY_DSN` env var. Disabled when DSN not set. `SentrySdk.CaptureException()` in GlobalExceptionFilter for manual capture with scope enrichment
- **Excel export pattern**: ClosedXML 0.104.2 in Application project. Use `XLWorkbook` → `Worksheets.Add()` → populate cells → `SaveAs(MemoryStream)` → return `byte[]`. Controller returns `File(bytes, contentType, fileName)`. Frontend uses `responseType: 'blob'` + `URL.createObjectURL()` for download
- **Frontend DTO alignment**: Frontend interfaces must match backend DTO field names exactly (camelCase). When backend returns `createdAt`, frontend uses `createdAt` — not custom Portuguese names
- **PDF export pattern**: QuestPDF 2024.12.2 in Application project. Set `QuestPDF.Settings.License = LicenseType.Community` in Program.cs. Service returns `byte[]`, controller returns `File(bytes, "application/pdf", fileName)`. Frontend uses `responseType: 'blob'` + `URL.createObjectURL()` for download
- **QuestPDF table header gotcha**: `table.Header()` callback param is NOT `TableDescriptor` — inline header cells directly in the lambda instead of passing to a typed helper
- **ToastService types**: 'success' | 'warning' | 'danger' | 'info' — use 'danger' not 'error'
- **Materialized view in unit tests**: `SkuProfitabilityViews` fails on SQLite — wrap queries in try-catch and return empty result gracefully
- **EF Core expression tree + optional params**: Record constructors with optional params (e.g., `string? X = null`) fail in `Select()` projections — use explicit `(Type?)null` instead
- **IEmailService**: Stub `NoOpEmailService` registered in both API and Worker. When US-059 adds real provider, swap in both `Program.cs` files
- **Worker cross-tenant queries**: Use `IgnoreQueryFilters()` in workers since there's no tenant context — filter by TenantId manually
- **Post-query enrichment pattern**: For data from materialized views/cross-entity lookups, fetch the main query first, then batch-lookup supplementary data and merge with `record with { }` syntax
- **Worker marketplace API access**: Worker needs its own DataProtection + ITokenEncryptionService + HttpClient("MercadoLivre") + keyed IMarketplaceAdapter registration (separate from API Program.cs). Add `Microsoft.AspNetCore.DataProtection` package to Worker .csproj
- **ML API resilience**: `Microsoft.Extensions.Http.Resilience` (Polly v8) via `.AddMercadoLivreResilience()` on the named HttpClient. Chain: Rate Limiter → Retry → Circuit Breaker. Extension in `Infrastructure/Marketplace/MercadoLivreResilienceExtensions.cs`
- **ProductVariantDto dual mapping**: `ProductVariantDto` is mapped in TWO places in `ProductService` — `GetVariantsAsync` (LINQ Select) AND `MapToVariantDto` helper. When adding new fields to ProductVariant, update BOTH.
