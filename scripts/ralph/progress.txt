## Codebase Patterns
- Angular CLI v21 with standalone components (no NgModules)
- All page components use `loadComponent` for lazy loading in app.routes.ts
- Working directory for Angular commands: `src/PeruShopHub.Web/`
- Build: `npx ng build`, Typecheck: `npx tsc --noEmit -p tsconfig.app.json`
- Proxy config in `proxy.conf.json` targets `http://localhost:5000` for `/api` routes
- Design tokens in `src/styles/tokens.css`, SCSS mixins in `src/styles/mixins.scss`
- Use `@use '../styles/mixins' as m;` then `@include m.mobile { ... }` in component SCSS
- Injectable root services go in `src/app/services/` with `{ providedIn: 'root' }`
- Shared pipes/components go in `src/app/shared/pipes/` and `src/app/shared/components/` with barrel exports via `index.ts`
- SCSS `@use` path from `src/app/shared/components/{name}/` to `src/styles/` is `../../../../styles/mixins`
- SCSS `@use` path from `src/app/pages/{name}/` to `src/styles/` is `../../../styles/mixins`
- Type re-exports need `export type { ... }` due to `isolatedModules` in tsconfig
- Lucide icon data type for `[img]` binding is `LucideIconData` from `lucide-angular` (not `object`)
- Toast container goes in app root (app.html), inject `ToastService` to show toasts from any component
- ng2-charts v10: `provideCharts(withDefaultRegisterables())` in app.config.ts, `BaseChartDirective` in component imports, Chart.js colors must be hardcoded hex (no CSS vars)
- For custom cell rendering (badges, icons, colored values), build a custom table instead of DataTableComponent
- Badge variant methods must return `BadgeVariant` type, not `string` — Angular strict template checking enforces this
- Route order: specific paths like `/produtos/novo` must come BEFORE parameterized `/produtos/:id`

## 2026-03-22 - US-003
- Created `ThemeService` as injectable root service at `src/app/services/theme.service.ts`
- Implements `currentTheme` signal with type `'light' | 'dark' | 'system'`
- `resolvedTheme` computed signal resolves 'system' to actual 'light'/'dark' based on `prefers-color-scheme`
- Loads preference from `localStorage('theme-preference')`, defaults to 'system'
- `setTheme()` updates signal, persists to localStorage, applies `data-theme` attribute
- `toggleTheme()` cycles: light → dark → system
- Listens to `matchMedia` change events for real-time OS theme changes
- Uses Angular `effect()` to apply theme whenever `resolvedTheme` changes
- Files changed: `src/app/services/theme.service.ts` (new)
- **Learnings for future iterations:**
  - Services go in `src/app/services/` directory
  - Angular 21 signals + `effect()` work well for reactive theme management
  - Theme applies by setting/removing `data-theme="dark"` on `document.documentElement` (matching tokens.css selectors)
